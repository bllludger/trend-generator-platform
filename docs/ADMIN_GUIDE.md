# Руководство по админ-панели

Как пользоваться админкой: разделы, типовые задачи. Вход — по логину и паролю из `.env` (`ADMIN_USERNAME`, `ADMIN_PASSWORD`). URL после запуска: http://localhost:3000 (или ваш домен).

---

## Разделы меню

### Дашборд

Сводка: метрики, счётчики (пользователи, задачи, платежи). Точный набор виджетов зависит от реализации страницы.

### Пользователи

- **Список пользователей** — поиск, фильтры (в т.ч. по статусу: забанен, подвешен).
- **Карточка пользователя** — детализация: баланс, сессии, лимиты (free_takes_used, copy_generations_used), реферальный код, трафик, заметки админа.
- **Типовые задачи:** выдать пакет пользователю (Grant pack), сбросить лимиты (free/copy, trial), разбан/снять suspend, отправить сообщение в Telegram. При необходимости — просмотр платежей и избранного пользователя.

### Задачи (Jobs)

Список задач генерации (legacy Job, не Take): статус (PENDING, SUCCEEDED, FAILED), тренд, пользователь, дата. Фильтр по статусу. Используется для отладки «генерация не пришла» и поиска упавших джобов.

### Платежи

- **Платежи** — список транзакций (Stars, ЮMoney, банк): пользователь, пакет, сумма, дата, статус. Просмотр и при необходимости возвраты (refund).
- **Пакеты** — настройка тарифов (Neo Trial, Neo Start, Neo Pro, Neo Unlimited): название, цена в Stars/рублях, лимиты снимков и HD, включение/выключение. От них зависят кнопки в магазине бота.
- **Банковский перевод** — настройки приёма оплаты переводом: реквизиты карты, комментарий для пользователя, лимиты; просмотр логов загруженных чеков и результатов распознавания.

### Реклама (Traffic)

Источники трафика и кампании (deep link `?start=src_<slug>_c_<campaign>`): настройка слагов и кампаний для атрибуции.

### Безопасность

- **Rate limit** — глобальный лимит запросов в час на пользователя; возможность включить обход для VIP.
- **Бан** — заблокировать пользователя (указать причину); бан отображается в боте при любом действии.
- **Suspend** — временная блокировка до указанной даты с причиной.
- **Список пользователей по статусу** — быстрый просмотр забаненных/подвешенных для снятия или эскалации.

### Аудит

Журнал действий: кто (админ/система), какое действие, над какой сущностью (user, trend, job, payment и т.д.). Просмотр при расследовании инцидентов и изменений настроек.

### Телеметрия

События продукта (product_events): воронки, кнопки, конверсии. Зависит от реализации страницы (графики, фильтры по событию/дате). Справочник событий — [TELEMETRY_EVENT_CATALOG.md](TELEMETRY_EVENT_CATALOG.md).

### Тренды

- **Тренды** — список трендов: название, тема, ЦА, включён/выключен, порядок. Редактирование промптов (сцена, перенос), негативный промпт, примеры изображений. От этого зависит, что видит пользователь в «Создать фото».
- **Тренды — аналитика** — статистика по выборам трендов (просмотры, выборы).
- **Автопостер** — настройки автопостинга трендов в канал (если включено); привязка к каналу через `POSTER_CHANNEL_ID`.

### Playground

Тестирование промптов и параметров генерации без влияния на прод: ввод промпта, загрузка фото, запуск генерации, просмотр результата. Для отладки мастер-промпта и трендов.

### Стиль копирования (Copy style)

Настройки сценария «Сделать такую же»: промпты анализа референса, модель, лимиты.

### Настройки

Общие настройки приложения: флаги функций, лимиты, URL каналов и т.д. (зависит от модели app_settings).

### Мастер промпт

Глобальный промпт генерации: системный префикс, негативный промпт, правила идентичности, безопасность, шаблон вывода. Редактирование влияет на все тренды. Подробнее — [PROMPT_ARCHITECTURE_AND_ADMIN_FLOW.md](PROMPT_ARCHITECTURE_AND_ADMIN_FLOW.md).

### Политика превью

Централизованные правила: когда показывать превью с watermark, когда сразу полный доступ; привязка к пакетам и free preview.

### Сообщения бота

Шаблоны текстов бота (приветствие, ошибки, подсказки): редактирование без деплоя кода.

### Рассылка

Отправка сообщения выбранным пользователям (broadcast) или одному пользователю (из карточки пользователя). Используется Celery-задача отправки в Telegram.

### Рефералы

Статистика реферальной программы: начисления бонусов 4K, список рефералов, при необходимости — заморозка сомнительных начислений.

### Склейка фото (Photo merge)

Включение/выключение и настройки сценария «Соединить фото» (2/3 лица).

### Очистка (Cleanup)

Админские операции очистки данных (отдельный сервис на порту 8001): запуск по кнопке, с авторизацией. Точный список операций — в интерфейсе и в `app/cleanup/`.

## Типовые сценарии

- **Пользователь не получил пакет после оплаты** — Платежи: проверить, есть ли платёж и сессия; при оплате по ссылке — проверить логи API (webhook ЮKassa) и при необходимости вручную активировать пакет через Grant pack в карточке пользователя.
- **Разбан / снять suspend** — Безопасность или карточка пользователя: снять флаги и при необходимости обновить причину/дату.
- **Выдать пакет вручную** — Пользователи → карточка пользователя → Grant pack (выбор пакета и подтверждение).
- **Изменить текст бота** — Сообщения бота: найти ключ шаблона и отредактировать текст.
- **Отключить тренд** — Тренды: снять галочку «Включён» или аналог.
- **Проверить, почему генерация упала** — Задачи: найти job по пользователю/дате, статус FAILED; Аудит — поиск по entity_type=job/take и действиям с ошибкой. Логи воркера — см. [RUNBOOK.md](RUNBOOK.md).
