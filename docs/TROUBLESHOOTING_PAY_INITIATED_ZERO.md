# «Нажата оплата» в воронке показывает 0

## Откуда берётся значение

- Воронка (Funnel) читает из таблицы **`product_events`**, событие **`pay_initiated`**.
- Событие пишется в боте при нажатии пользователем «Оплатить» (Stars, YooMoney, YooMoney link, разблокировка, buy legacy, upgrade).

## Почему может быть 0

1. **Бот не обновлён** — контейнер бота запущен со старым образом без вызовов `track("pay_initiated", ...)`.
2. **Ошибка при записи** — `ProductAnalyticsService.track()` падает (например FK по `user_id`), исключение логируется, событие не сохраняется.

## Что сделать

### 1. Убедиться, что бот с обновлённым кодом запущен

```bash
# Пересобрать и перезапустить бот
docker compose build bot && docker compose up -d --force-recreate bot
```

После этого кто-то должен нажать оплату (Stars или YooMoney и т.д.) — в течение 7 дней событие попадёт в воронку за «7 д.»

### 2. Проверить логи бота при нажатии оплаты

При ошибке записи в логах будет строка вида:

```text
product_analytics track failed: event=pay_initiated user_id=... error=...
```

Если такая строка есть — смотреть `error=...` (часто FK на `users.id` или недоступность БД).

### 3. Проверить, есть ли события в БД

На сервере (или из любого клиента к БД):

```bash
docker compose exec db psql -U "${POSTGRES_USER:-trends}" -d "${POSTGRES_DB:-trends}" -c "
  SELECT event_name, COUNT(*), MAX(timestamp) 
  FROM product_events 
  WHERE event_name = 'pay_initiated' 
  GROUP BY event_name;
"
```

- Если выборка пустая — события не писались (старый бот или ошибка в `track()`).
- Если строки есть — воронка должна показывать этот счётчик; проверь окно «7 д.» и кеш/обновление страницы.

### 4. После деплоя

Сделать тестовое нажатие оплаты (можно не доводить до реальной оплаты) и снова проверить воронку через 1–2 минуты или запрос выше.
