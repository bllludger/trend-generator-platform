# Правила preview pipeline

## Форматы

- **PNG** — только как original или промежуточный asset внутри системы. Провайдеры (Gemini и др.) могут возвращать PNG; сохраняем без изменений.
- **JPEG / WebP** — формат preview. Наружу до оплаты отдаётся только preview в web-формате (по настройкам политики превью).

## Поток данных

1. **Original** сохраняется в исходном формате провайдера (без конвертации).
2. **Preview** строится один раз через `PreviewService`: original → resize → watermark → encode.
3. Preview сохраняется в JPEG или WebP в зависимости от настроек (Политика превью в админке).
4. При повторной отправке в Telegram используется уже готовый preview-файл, без повторной перекодировки.

## MIME при отправке в Telegram

MIME определяется по **фактическому расширению** отправляемого файла (preview = .webp или .jpg). Реализация: `TelegramClient._mime_for_photo_path(path)` в `app/services/telegram/client.py`.

## Зачем

- Не зависеть от формата ответа провайдера.
- Не отдавать PNG как бесплатное превью.
- Уменьшить размер файлов и нагрузку на доставку.
- Сохранить скорость за счёт одноразовой сборки preview.
