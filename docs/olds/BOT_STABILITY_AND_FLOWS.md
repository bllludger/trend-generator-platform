# Стабильность Telegram-бота: команды и флоу

Отчёт по анализу устойчивости всех точек входа и сценариев. Цель: пользователь всегда получает ответ (успех или сообщение об ошибке), callback-запросы не «висят».

## 2. Флоу «Создать фото» (Session/Take)

```
[Кнопка «Создать фото»] → request_photo (ЦА)
    → [Выбор ЦА] audience: → on_audience_selected (запрос фото)
    → [Фото] handle_photo_step1 / handle_photo_as_document_step1 (сохранение, тематики)
    → [Тема] theme:* → select_theme_or_theme_page (тренды)
    → [Тренд] trend:* → select_trend_or_idea (формат или «Своя идея»)
    → [Своя идея] F.text в waiting_for_prompt → ввод текста
    → [Формат] format:* → select_format_and_generate (создание Take, Celery)
    → [Генерация] generate_take (воркер): прогресс → 3 варианта или ошибка
    → [Вариант] choose:* → избранное + paywall или «Ещё снимок»
```

- **Проверки по шагам:** сессия/лимиты, фото в state, тренд доступен, размер файла, идемпотентность (format).
- **Ответы при ошибках:** везде callback.answer(...) или message.answer(...); воркер при падении шлёт «Генерация не удалась» / «Произошла ошибка при генерации».
- **Состояния:** при «неправильном» сообщении unknown_message даёт подсказку под текущий шаг (waiting_for_audience, waiting_for_photo, waiting_for_trend, waiting_for_format, waiting_for_prompt).

## 3. Флоу «Сделать такую же» (Copy style)

```
[Кнопка «Сделать такую же»] → start_copy_flow (референс)
    → [Референс фото/документ] → сохранение, Vision, тематики
    → [1 или 2 фото] copy_photos:1/2
    → [Своё фото 1] (и при 2 фото — второе)
    → формат → select_format_and_generate (тот же путь, copy_reference_path)
```

- Ошибки сохранения/файла/Vision обрабатываются, пользователю показывается сообщение.
- Обработчики документов при `doc is None` отвечают «Ошибка. Попробуйте позже.»

## 8. Перегенерация (Job flow)

```
[regenerate:job_id] → regenerate_same
    → проверка job, тренда, исходных фото, лимитов
    → Celery generation_v2.generate_image
```

- В боте: все ветки с callback.answer (job не найден, тренд недоступен, нет фото и т.д.).
- В воркере generation_v2: при необработанном исключении в чат отправляется «Произошла ошибка при генерации. Попробуйте ещё раз.»

---

## 9. Глобальный обработчик ошибок

- `dp.errors.register(on_error)`: логирует исключение; **не** отправляет сообщение пользователю.
- Все критические хендлеры имеют локальный try/except и отправляют пользователю сообщение об ошибке, чтобы при любом сбое пользователь не оставался без ответа.

---

## 10. Рекомендации по поддержке

1. При добавлении новых хендлеров: всегда вызывать `callback.answer()` во всех ветках (в т.ч. при раннем return) и оборачивать логику в try/except с ответом пользователю.
2. Воркеры (generate_take, deliver_hd, generation_v2): при фатальной ошибке отправлять в `status_chat_id` сообщение об ошибке, если он передан.
3. Состояния FSM: при добавлении нового состояния добавлять для него ветку в `unknown_message`, чтобы при «неправильном» вводе показывать подсказку по текущему шагу.
