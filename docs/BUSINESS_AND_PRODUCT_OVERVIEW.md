# Описание продукта и бизнес-логики (Trend Generator / Nano Banana)

Документ описывает продукт с точки зрения бизнеса и пользовательского опыта — без технических деталей. Предназначен для продуктологов, маркетинга и стейкхолдеров.

---

## 1. Назначение продукта

**Trend Generator** — это Telegram-бот, который по фотографии пользователя создаёт стилизованные изображения в выбранном «тренде» (визуальный стиль, сцена, настроение). Пользователь загружает своё фото, выбирает тему и тренд, после чего получает несколько вариантов изображения. Часть функционала монетизируется: бесплатно доступны ограниченные возможности, за плату — фотосессии с большим числом образов и получение изображений в высоком разрешении (4K) без водяного знака.

**Ключевая ценность:** превратить одно фото в серию стилизованных «образов» по готовым трендам (например, ретро, кино, арт) с возможностью забрать выбранные кадры в HD.

---

## 2. Целевая аудитория и сценарии

- **Женщины** — основная ЦА: тренды и темы подобраны под женские портреты.
- **Мужчины** — часть трендов доступна для мужской ЦА.
- **Пары** — сценарии для пар (два лица в кадре).

При первом действии «Создать фото» пользователь выбирает целевую аудиторию (Женщина / Мужчина / Пара). От этого зависит набор доступных тем и трендов: каждые тема и тренд могут быть привязаны к одной или нескольким ЦА.

---

## 3. Основные сценарии использования

### 3.1. «Создать фото» (основной флоу)

1. Пользователь нажимает **«Создать фото»** (или сразу отправляет фото).
2. Выбирает **целевую аудиторию** (Женщина / Мужчина / Пара).
3. Отправляет **одно или несколько фото** (в зависимости от тренда).
4. Выбирает **тему** (категория трендов, например «Кино», «Ретро»).
5. Выбирает **тренд** (конкретный стиль) или **«Своя идея»** — ввод текстового описания.
6. Выбирает **формат** кадра (1:1, 16:9, 4:3 и т.д.).
7. Запускается генерация. Пользователь получает **три варианта** (A, B, C) одного снимка.
8. Выбирает понравившийся вариант — он попадает в **избранное**. При необходимости может запросить **HD** (4K без водяного знака) в рамках купленной фотосессии или после оплаты.

Весь путь от фото до результата — **одна «фотосессия»** (сессия). Количество снимков и право на HD задаются купленным тарифом.

### 3.2. «Сделать такую же» (копирование стиля)

Пользователь хочет повторить стиль понравившегося изображения:

1. Нажимает **«Сделать такую же»**.
2. Отправляет **референс** — фото-образец стиля (например, скриншот или чужую картинку).
3. Система анализирует референс и формирует описание стиля.
4. Пользователь указывает количество лиц (1 или 2) и загружает **свои фото**.
5. Выбирает формат — получает три варианта по тому же сценарию, что и в «Создать фото».

На этот сценарий действует отдельный лимит бесплатных использований (один раз на аккаунт).

### 3.3. Коллекции (outcome-пакеты)

Часть тарифов продаётся как **коллекции**: фиксированный набор трендов (плейлист). Обещание пользователю: «всего N образов, выбери до M в HD, забери альбомом».

- После покупки коллекции пользователь **один раз** отправляет **одно фото**.
- По этому фото по очереди генерируются все образы из плейлиста (каждый шаг — один тренд, три варианта).
- Пользователь по каждому шагу выбирает вариант в избранное и может отметить его для HD.
- В конце может **забрать все выбранные HD одним альбомом** (или по одному).

Лимиты: сколько превью всего, сколько HD можно выбрать — задаются пакетом (тарифом). Счётчики «осталось снимков», «HD осталось», «в избранном» отображаются в интерфейсе.

## 4. Монетизация и тарифы

### 4.1. Продуктовая лестница (SKU)

В продаже только четыре тарифа:

| Тариф | Описание для пользователя | Роль |
|-------|----------------------------|------|
| **Trial** | Один снимок для пробы | Входной, один раз на аккаунт; после него предлагается доплата до Neo Start / Neo Pro. |
| **Neo Start** | 10 образов + 10 HD без watermark | Базовый пакет. |
| **Neo Pro** | 40 образов + 40 HD без watermark | Основной денежный пакет (хит, «Best value»). |
| **Neo Unlimited** | 120 образов + 120 HD без watermark | Максимальный пакет (anchor). |

Цены задаются в **Telegram Stars** (и дублируются в рублях для отображения). Trial — самый дешёвый вход; после его использования показывается апсейл: «99⭐ уже учтены, доплати до Neo Start или Neo Pro».

### 4.2. Где пользователь платит

- **«Купить тариф»** — главное меню; открывается экран выбора из четырёх тарифов. После оплаты создаётся активная фотосессия с соответствующими лимитами (число снимков, число HD).
- **Paywall после бесплатного превью** — в сценарии с одним бесплатным снимком после выбора варианта показывается экран оплаты (максимум три кнопки: Trial, текущая коллекция/тариф, Creator/макс. пакет).
- **Апгрейд сессии** — при исчерпании лимита или по желанию пользователь может докупить тариф; зачёт уже уплаченных Stars (например, после Trial) учитывается.
- **Оплата переводом** — кроме Stars поддерживается оплата «переводом на карту»: пользователь платит по реквизитам и загружает чек; после проверки начисляется соответствующий тариф.

Никакие другие пакеты (старые «session», «studio», разовые «токены» и т.д.) в интерфейсе не показываются и не принимаются.

### 4.3. Баланс и лимиты

- **Баланс фотосессии** — «осталось снимков: X из Y», «4K без watermark: Z». Это лимиты текущей активной сессии (привязаны к купленному тарифу), а не «токены» на аккаунт.
- После каждой генерации пользователю показывается напоминание: баланс + призыв выбрать фотосессию (тарифы с кнопками). То же — при явном запросе «Пополнить баланс» и при исчерпании лимита снимков.

## 5. Избранное и доставка HD

- **Избранное** — выбранные пользователем варианты (по одному на снимок) накапливаются в «Избранном» в рамках сессии.
- Пользователь может **отметить** часть из них для получения в **HD** (4K без водяного знака), в пределах лимита HD по тарифу.
- **Доставка HD** — по кнопке можно запросить доставку одного выбранного HD или «забрать альбомом» все отмеченные для HD. Доставка идёт в чат (файлы в высоком разрешении).
- **Гарантия качества:** HD — это тот же образ, что превью, только в большем разрешении (без перерисовки), чтобы «что выбрал — то и получил».

Если доставка HD не произошла в заданный срок (SLA) или произошла постоянная ошибка — пользователю автоматически возвращается право на один HD (компенсация). Есть кнопка «Проблема с HD» для обращения в поддержку с привязкой к конкретному снимку.

---

## 6. Доверие, согласие и данные

- Перед первым созданием фото запрашивается **согласие** пользователя (кнопка «Принимаю»).
- В боте доступна команда/кнопка **«Удалить мои данные»**: по запросу данные пользователя и связанные файлы удаляются в соответствии с политикой.
- **Компенсации** при сбоях HD (авто-возврат HD или обработка по обращению) повышают доверие и снижают конфликты.

---

## 7. Реферальная программа

- Пользователь имеет **реферальный код** и может приглашать друзей.
- При оплате приглашённым тарифа из продуктовой лестницы рефереру начисляются **бонусные кредиты 4K** (по лестнице в зависимости от суммы покупки).
- Начисление может быть с задержкой (холд) и в рамках лимитов (дневной/месячный). Админка позволяет просматривать статистику рефералов и бонусов, а также замораживать сомнительные начисления.

---

## 8. Админка и управление контентом

Админ-панель (веб-интерфейс) предназначена для управления продуктом и контентом.

### 8.1. Контент и генерация

- **Темы** — категории трендов (название, эмодзи, порядок, целевые аудитории: женщины, мужчины, пары).
- **Тренды** — визуальные стили внутри тем: название, описание, промпты для генерации, негативные промпты, стилевые пресеты, привязка к ЦА. Для каждого тренда можно загрузить пример результата и референс стиля.
- **Мастер-промпт и политика переноса** — глобальные настройки текстов, которые участвуют в сборке запроса к нейросети (перенос личности, композиция, безопасность). Редактируются в разделе настроек.
- **Промпт по трендам (Playground)** — для продвинутых трендов можно задать секции промпта и параметры генерации (модель, размер, температура, seed) для воспроизводимости и A/B-тестов.
- **«Сделать такую же»** — отдельные настройки: промпты и модель для анализа референсного фото.

### 8.2. Тарифы и оплата

- **Пакеты (тарифы)** — создание и редактирование тарифов: название, цена в Stars, лимиты (число снимков, число HD), тип (сессия, коллекция). Для коллекций задаётся плейлист (список трендов), лейбл, лимит избранного, SLA по HD, пакеты для апсейла после покупки.
- **Платежи** — просмотр списка платежей и статистики, возвраты (refund).
- **Оплата переводом** — настройки реквизитов и приёма чеков, просмотр логов загруженных чеков.

### 8.3. Пользователи и модерация

- **Пользователи** — поиск, просмотр профиля, балансы, флаги (VIP, тестер и т.д.), заметки.
- **Безопасность** — бан, разбан, временная блокировка (suspend), смена лимита запросов в час, назначение модератора (обход лимитов). Сброс лимитов бесплатных генераций.
- **Аудит** — лог значимых действий (создание задач, оплаты, смена настроек и т.д.).

### 8.4. Сессии, снимки, компенсации

- **Сессии** — просмотр активных и завершённых сессий пользователей (тариф, лимиты, прогресс по плейлисту для коллекций).
- **Задачи (Jobs)** — просмотр задач генерации и их статусов.
- **Компенсации** — список и статистика выданных компенсаций (HD при сбоях).

### 8.5. Коммуникация и телеметрия

- **Рассылка** — превью и отправка сообщений пользователям (broadcast).
- **Шаблоны сообщений** — редактирование текстов, которые бот отправляет пользователям (приветствие, ошибки, подсказки).
- **Телеметрия и метрики** — воронки (коллекция старт, превью готово, оплата, завершение коллекции, доставка HD, отвалы по шагам), метрики по продукту (например, конверсия Trial, средние Stars на платящего).

### 8.6. Прочее

- **Рефералы** — статистика и управление бонусами (в т.ч. заморозка).
- **Очистка** — запуск очистки временных файлов по правилам продукта (без удаления результатов генераций и нужных для работы данных).

---

## 9. Ключевые метрики и события (телеметрия)

Для анализа продукта фиксируются события, например:

- **collection_start** — старт коллекции (после покупки).
- **take_previews_ready** — по снимку готовы три превью (A/B/C).
- **pay_success** — успешная оплата.
- **collection_complete** — коллекция завершена (все шаги плейлиста пройдены).
- **hd_delivered** — доставлен HD пользователю.
- **collection_drop_step** — отвал на шаге (незавершённые сессии).
- **paywall_variant_shown** — какие кнопки paywall показаны.

По ним считаются воронки и продуктовые метрики в админке.

## 10. Итоговая логика продукта (кратко)

1. Пользователь создаёт фото через **«Создать фото»** или **«Сделать такую же»**, выбирая ЦА, тему, тренд (или свою идею) и формат.
2. По одному фото генерируются **три варианта**; пользователь выбирает один в избранное и при желании отмечает его на HD в рамках лимитов сессии.
3. **Фотосессия** задаётся купленным тарифом: Trial (1 снимок), Neo Start (10), Neo Pro (40), Neo Unlimited (120) — с соответствующим числом HD.
4. В **коллекциях** одно фото проходит по плейлисту трендов; пользователь по каждому шагу выбирает вариант и в конце забирает выбранные HD альбомом или по одному.
5. **Монетизация** — только через четыре тарифа; после генераций и при исчерпании лимита показывается экран «баланс + выбор фотосессии» с кнопками тарифов.
6. **Доверие** обеспечивается согласием, удалением данных по запросу и компенсациями при сбоях HD. **Реферальная программа** поощряет приглашения бонусными HD-кредитами.
7. **Админка** управляет темами, трендами, промптами, тарифами, платежами, пользователями, компенсациями, рассылками и метриками.

Документ можно обновлять при изменении продуктовой логики и тарифов.
