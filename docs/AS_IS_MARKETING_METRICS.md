# AS-IS: Метрики для маркетинга

**Цель:** дать маркетингу понятную картину воронки, аудитории, трендов и каналов (в т.ч. реферальная программа) на основе данных из БД и админ-API.

## Слайд 2 — Воронка конверсии

Действия пользователя от входа в сценарий до выдачи HD. Счётчики за выбранный период (по умолчанию 7 дней):

| Этап воронки | Действие в системе | Поле в ответе API |
|--------------|-------------------|---------------------|
| Старт коллекции | Пользователь начал сценарий «создать фото» | `collection_start` |
| Превью готовы | Получены 3 варианта снимка | `take_previews_ready` |
| Оплата | Успешная оплата (Stars) | `pay_success` |
| Коллекция завершена | Сессия/коллекция завершена | `collection_complete` |
| HD доставлено | Пользователь получил HD-файл | `hd_delivered` |

**API:** `GET /admin/telemetry/product-metrics?window_days=7`  
**Ответ:** `funnel_counts` — объект с ключами указанных действий и количеством событий.

По этим шагам можно считать конверсию между этапами (например, pay_success / take_previews_ready).

## Слайд 3 — Аудитория: DAU, WAU, MAU, липкость

| Метрика | Значение | Как использовать |
|--------|----------|-------------------|
| **DAU** | Уникальные пользователи с активностью (Job или Take) за последние 24 ч | Ежедневная активность |
| **WAU** | За 7 дней | Недельная активность |
| **MAU** | За 30 дней | Месячная аудитория |
| **Stickiness, %** | DAU / MAU × 100 | Насколько часто пользователи возвращаются в течение месяца |

**API:** `GET /admin/telemetry/product-metrics?window_days=7`  
Поля: `dau`, `wau`, `mau`, `stickiness_pct`.

## Слайд 4 — Когорты и рост

**Новые пользователи по месяцам (последние 12 месяцев)**  
Позволяет смотреть рост базы и сезонность.

**API:** `GET /admin/users/analytics?time_window=30`  
Ответ: `cohorts` — массив `{ "month": "YYYY-MM", "count": N }`.

**Новые пользователи по дням (последние 14 дней)**  
Для оперативного графика регистраций.

Ответ того же эндпоинта: `growth_list` — массив `{ "date": "YYYY-MM-DD", "new_users": N }`.

## Слайд 5 — Тренды: что генерируют и что выбирают

Тренд — стиль/формат генерации (например, «Неон», «Ретро»). Для маркетинга важно:
- какие тренды чаще всего **генерируют** (Take и Job);
- по каким трендам пользователи чаще **выбирают вариант** (добавляют в избранное).

**API:** `GET /admin/telemetry?window_hours=24` (или 168 для недели).

В ответе массив **trend_analytics_window**. По каждому тренду:
- `trend_id`, `name`, `emoji`
- `jobs_window` — задач (Job) за окно
- `succeeded_window`, `failed_window` — успех/провал Job
- `takes_window`, `takes_succeeded_window`, `takes_failed_window` — снимки (Take)
- **`chosen_window`** — сколько раз пользователь выбрал вариант (добавил в избранное) по этому тренду

Интерпретация: высокий `chosen_window` при высоком `takes_window` — тренд и востребован, и нравится. Низкий chosen при высоком takes — много генераций, но мало выбора (можно улучшать контент или UX).

## Слайд 6 — Платежи и средний чек (AOV)

| Метрика | Описание | API |
|--------|----------|-----|
| **Количество успешных оплат** | События pay_success за период | product-metrics → `total_pay_success_count` |
| **Доля покупок Trial** | % покупок пакета trial | `share_trial_purchases`, `trial_purchases_count` |
| **Средний чек в Stars** | На один pay_success | `avg_stars_per_pay_success` |
| **Выручка за период** | По таблице payments (completed) | `GET /admin/payments/stats?days=30` → `total_stars`, `revenue_usd_approx`, `revenue_rub_approx` |
| **Уникальные покупатели** | За период | `unique_buyers` |
| **Разбивка по пакетам** | Какие пакеты покупают | `by_pack` (pack_id, count, stars) |

Для маркетинга: AOV, доля Trial и структура по пакетам помогают оценивать эффективность предложений и цен.

## Слайд 7 — Реферальная программа

| Метрика | Описание | API |
|--------|----------|-----|
| **Приведённые пользователи** | Всего зарегистрировавшихся по реферальной ссылке | `GET /admin/referrals/stats` → `total_attributed` |
| **Бонусы по статусам** | pending / available / spent / revoked | `by_status` |
| **Кредиты HD** | Суммарно в pending, available, spent | `credits` |

Маркетинг может оценивать объём привлечения через рефералов и «стоимость» бонусов (credits).

## Слайд 8 — Сегменты по активности

Распределение пользователей по количеству **задач (Job)** за выбранный период (например, 7 дней):

| Сегмент | Поле в API | Интерпретация |
|---------|------------|----------------|
| 1 задача | `jobs_per_user_distribution["1"]` | Разовое использование |
| 2–5 задач | `["2_5"]` | Низкая вовлечённость |
| 6–10 | `["6_10"]` | Средняя |
| 11–20 | `["11_20"]` | Высокая |
| 21+ | `["21_plus"]` | Очень активные |

**API:** `GET /admin/telemetry/product-metrics?window_days=7` → `jobs_per_user_distribution`.

Дополнительно в **GET /admin/users/analytics** есть **activity_segments**: «Без задач», «1–5 задач», «6–20», «21+» — по пользователям за окно. Удобно для сообщений «холодным» vs «тёплым».

## Слайд 9 — Тематики и целевая аудитория (из БД)

В БД у **тем (themes)** и **трендов (trends)** есть поле **target_audiences** (например, `["women"]`, `["men"]`, `["couples"]`). Это не метрики, а настройки контента:
- какие темы/тренды показывать каким сегментам (по полу/типу пары).

Для маркетинга: понимание, под какую ЦА настроен контент; при необходимости можно сопоставлять с поведением (например, конверсия по трендам с разными target_audiences через общие отчёты по трендам).

## Слайд 10 — Где смотреть: сводка API для маркетинга

| Задача | Эндпоинт | Ключевые поля |
|--------|----------|----------------|
| Воронка | GET /admin/telemetry/product-metrics?window_days=7 | funnel_counts |
| DAU/WAU/MAU, липкость | Тот же | dau, wau, mau, stickiness_pct |
| AOV, Trial, распределение по активности | Тот же | avg_stars_per_pay_success, share_trial_purchases, jobs_per_user_distribution |
| Когорты и рост | GET /admin/users/analytics?time_window=30 | cohorts, growth_list |
| Тренды (генерации + выбор варианта) | GET /admin/telemetry?window_hours=168 | trend_analytics_window |
| Платежи и выручка | GET /admin/payments/stats?days=30 | total_payments, total_stars, by_pack, unique_buyers |
| Рефералы | GET /admin/referrals/stats | total_attributed, by_status, credits |
| Сегменты по задачам | GET /admin/users/analytics | activity_segments |

Все запросы — авторизованный админ-API (JWT).

## Слайд 11 — Рекомендации по использованию

1. **Еженедельно:** воронка (funnel_counts), DAU/WAU/MAU, stickiness, AOV и доля Trial.  
2. **Ежемесячно:** когорты (cohorts), выручка и разбивка по пакетам, реферальная программа.  
3. **При запуске кампаний/изменении контента:** тренды (trend_analytics_window, chosen_window) и сегменты активности (jobs_per_user_distribution, activity_segments).  
4. **Целевая аудитория:** смотреть настройки target_audiences у тем и трендов в админке и сопоставлять с отчётами по трендам.

Этого набора достаточно, чтобы описывать AS-IS состояние сервиса для маркетинга и принимать решения по каналам, контенту и предложениям.
